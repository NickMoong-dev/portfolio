Case 1

로어 사서 RAG 파이프라인 설계

① 문제 정의
유저의 소설 설정 데이터가 방대해져서 LLM 컨텍스트 윈도우를 초과. 장편 연재물에서 수백 개의 설정 간 충돌이 발생.
② 의사 결정 기준
Orama(인메모리) vs pgvector 트레이드오프를 AI와 토론. 제약조건: ❶ 오프라인 설치형 데스크탑 앱 ❷ 추가 서버 설치 불가 ❸ PC 사양 최소화. → Frontend RAG(Orama) + Backend RAG(SQLite Vector fallback) 하이브리드 채택.
③ 구현 & 검증 결과
사서 AI가 유저의 원고를 분석하여 관련 설정을 실시간 제안. harness.ps1 자동화 검증 통과. Vitest 73개 테스트 케이스 전체 패스. 백과사전 DB 기반 컨텍스트 주입으로 설정 충돌 방지 실증.
Case 2

Electron IPC 스트리밍 병목 해결

① 문제 정의
로컬 LLM 스트리밍 시 초당 수십 번의 Zustand 상태 업데이트 → React 리렌더링 폭주 → UI 멈춤 현상.
② 의사 결정
디바운싱 → 스트리밍이 끊겨 보임. throttle → 반응성 저하. → No-Store-Actions 패턴 채택: 로컬 Ref 누적 + requestAnimationFrame 배치 렌더링 + 완료 시점에만 Store 커밋.
③ 검증 결과
60fps 부드러운 스트리밍 UX 달성. 리렌더링 횟수 95% 감소. 텀블벅 89명 배포 후 UI 멈춤 리포트 0건.
Case 3

MCP 서버 구현 — Claude Desktop 연동

① 문제 정의
유저가 Aether AI 내부에서 만든 커스텀 AI 봇(Gems)을 외부 AI 도구(Claude Desktop, Cursor)에서도 호출하고 싶다는 요구사항.
② 의사 결정
REST API vs MCP(Model Context Protocol). MCP는 Claude Desktop 네이티브 지원 + 표준화된 프롬프트/툴 노출 규격. → MCP SDK 기반 Stdio 서버 채택. @modelcontextprotocol/sdk 사용.
③ 구현 결과
aetherMCPServer.ts — ListPrompts, GetPrompt, ListTools, CallTool 4개 핸들러 구현. 로컬 파일시스템에서 커스텀 봇(Gems)을 읽어와 MCP 표준 프롬프트로 자동 배포. Claude Desktop에서 Aether 세계관 설정 호출 검증 완료.
